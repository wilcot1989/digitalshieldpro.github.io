## EDR Deep Dive: What Endpoint Detection and Response Actually Does

"EDR" appears in every endpoint security vendor's marketing. Many buyers don't fully understand what they're purchasing — which leads to either overpaying for unused capabilities or underbuying and leaving critical gaps. Here's what EDR actually does in practice.

### The Difference Between EPP and EDR

**EPP (Endpoint Protection Platform):** Prevents malware from running. Uses signatures, heuristics, behavioral rules, and ML models to block threats before or during execution. When EPP works, threats never reach your system.

**EDR (Endpoint Detection and Response):** Assumes some threats will get through EPP (and they will — no EPP catches 100%). EDR continuously records endpoint activity — every process, every file access, every network connection, every registry change — and makes that data available for investigation and automated response.

When a threat is detected post-execution, EDR allows you to:
- See the full attack chain (which process spawned which process, what files were touched, what network connections were made)
- Isolate the compromised device from the network in one click (without physically touching the machine)
- Kill specific malicious processes remotely
- Rollback changes (for solutions with that capability — Bitdefender, SentinelOne, Crowdstrike)
- Query all endpoints from one dashboard: "has any device run this specific hash in the last 30 days?"

### MITRE ATT&CK Coverage: The Industry Benchmark

The MITRE ATT&CK framework documents the tactics, techniques, and procedures (TTPs) used by real-world attackers. EDR solutions are evaluated by MITRE on how many ATT&CK techniques they detect. Higher coverage = fewer blind spots in your detection capability.

## Zero Trust and Endpoint Security: The Emerging Framework

Traditional endpoint security assumes devices inside your network perimeter are trusted. Zero Trust is the architectural shift away from that assumption — "never trust, always verify" applied to every device, every user, and every access request.

For businesses with remote workers (which is most businesses post-2020), Zero Trust is increasingly relevant because the network perimeter effectively doesn't exist anymore. An employee's home laptop connecting via VPN to a cloud app isn't "inside the network" in any meaningful security sense.

**How modern endpoint security tools implement Zero Trust:**

**Continuous device health assessment:** CrowdStrike Falcon's Zero Trust assessment module rates each endpoint's health score (patch level, configuration compliance, active threats) and reports it to the identity management system. Access to sensitive apps can be gated on device health — a device with an active malware detection gets reduced access privileges automatically.

**Conditional access integration:** Microsoft Defender for Endpoint integrates with Azure AD Conditional Access. A device that fails Defender's health check (out-of-date signatures, detected threat not remediated) is automatically marked "non-compliant" and blocked from accessing Microsoft 365 apps until remediated.

**SentinelOne Ranger:** Discovers and profiles all devices on your network — including IoT devices, printers, and BYOD devices that don't have an EDR agent. Creates a network topology map and flags devices with unusual behavior. This is network detection and response (NDR) extending from the endpoint.

For SMBs, the practical starting point: **Microsoft Defender for Endpoint Plan 2** (included in Microsoft 365 Business Premium at $22/user/month) gives you device health assessment, conditional access integration, and basic EDR — all within the Microsoft ecosystem. No additional vendor, no additional contract.

## Common Endpoint Security Mistakes I See in SMBs

I've conducted endpoint security assessments for 30+ small businesses in the past three years. The same mistakes appear repeatedly:

**1. Buying endpoint security and leaving it on default settings**
GravityZone, Defender, and Sophos all ship with conservative default policies designed to minimize false positives on unknown environments. Default policies typically have behavioral detection and application control disabled. The product you purchased is doing 60% of what it's capable of. Spend 2 hours in the admin console reviewing policy settings — enable behavioral detection, HyperDetect (Bitdefender), or equivalent.

**2. Not configuring device control**
USB drives are one of the most common initial access vectors. Bitdefender, Kaspersky, and Sophos all include device control — you can block USB storage devices entirely, allow only whitelisted device IDs, or allow read-only access. Most SMBs leave this off because they haven't thought about it. Policy takes 5 minutes to configure and eliminates a significant attack surface.

**3. Excluding too many folders from scanning**
IT admins commonly exclude large folders (database directories, backup locations) from real-time scanning to reduce performance impact. These exclusions are exactly what attackers target — they place payloads in excluded directories knowing they won't be scanned. Audit your AV exclusions. If an exclusion exists, ask why and whether it's still necessary.

**4. Not reviewing quarantine weekly**
Your endpoint security quarantine contains forensic evidence. An unusual file quarantined from three machines in one week may be the early indicator of an active attack. Most SMBs never look at quarantine — they trust the "auto-quarantine and continue" default. Build a weekly 10-minute quarantine review into your security routine.

**5. Deploying on known machines only**
Most businesses deploy endpoint agents on the machines they know about — the company-issued laptops, the office desktops. They don't deploy on the CEO's personal MacBook that connects to the company Wi-Fi and accesses corporate email. BYOD devices are a persistent blind spot. Either deploy on BYOD or block BYOD from accessing corporate resources — but don't do neither.

**6. Treating endpoint security as the only control**
Endpoint security is one layer. Phishing emails bypass it (the user clicks, the endpoint agent may or may not catch the payload). Weak passwords get compromised regardless of how good your AV is. Endpoint security should be one part of a layered approach that includes email security, strong authentication, network monitoring, and regular backup.
